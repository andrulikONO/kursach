# Как проверить функционал

## Подготовка

1. Запуск по **`DOCKER.md`** или локально: PostgreSQL + `php -S` для `backend/public` + `npm run dev` для фронта.
2. В Docker админ создаётся при старте `backend` автоматически; локально один раз: **`php backend/scripts/seed_admin.php`** из корня проекта (из каталога с настроенным `backend/.env`).

## Роли

| Роль | Как получить |
|------|----------------|
| **Гость** | Не входить в систему (нет `localStorage.demoAuth`) |
| **Пользователь** | Регистрация на `/register` |
| **Администратор** | Скрипт `seed_admin.php`, вход логином `admin` |

Важно: в заголовке `Authorization` клиент передаёт только **`Demo user:<id>`**. Роли и блокировка **всегда** подгружаются из БД — подделать `admin` в заголовке нельзя.

## Сценарии

### 1. Гость

- Откройте главную `/` — виден каталог объявлений (`GET /api/products`).
- Откройте карточку объявления — видны поля, в т.ч. телефон из `contact_phone`, если он был сохранён при публикации.
- Страница «Подача объявления» `/new` должна предложить войти.
- `GET /api/me` без авторизации → **401**.

### 2. Регистрация и телефон

- `/register` — заполните все поля, **телефон** в формате **+7XXXXXXXXXX** (или 10 цифр без кода страны — нормализуется).
- После успеха — профиль `/profile`: в настройках отображаются данные из **`GET /api/me`**.
- `/new` — поле «Телефон в объявлении» только для чтения, значение из профиля; при отправке формы бэкенд копирует телефон в **`products.contact_phone`**.

### 3. Пользователь и объявления

- Войдите (`/login` или модальное окно) логином **или** email.
- Создайте объявление на `/new`.
- В каталоге на карточке отображается **📞 и телефон** (если есть).
- В профиле вкладка «Мои объявления» — данные из **`GET /api/my/products`**.

### 4. Блокировка

- Войдите как **admin**, откройте **`/admin`**, нажмите «Заблокировать» у тестового пользователя.
- Выйдите, войдите под заблокированным пользователем.
- Попытка **`POST /api/products`** → ответ **403** с текстом про блокировку; на `/new` показывается предупреждение.

### 5. Администратор

- Вход под `admin` / пароль из `seed_admin`.
- В шапке появляется ссылка **«Админ»** → `/admin`: список пользователей, блокировка / разблокировка.
- На странице объявления **`/product/:id`** — кнопка **«Удалить объявление (админ)»** (`DELETE /api/products/:id`, мягкое удаление: `status = deleted`).

### 6. API вручную (опционально)

```bash
# здоровье
curl -s http://localhost:8080/api/health

# каталог (гость)
curl -s http://localhost:8080/api/products

# вход
curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"identifier\":\"admin\",\"password\":\"Admin123!\"}"

# профиль (подставьте заголовок из ответа login: Demo user:N)
curl -s http://localhost:8080/api/me -H "Authorization: Demo user:1"
```
